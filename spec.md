# [Spec.md] AI 기반 모바일 근로계약서 서비스: Sik-Contract

## 1. 서비스 개요

- **서비스명**: 식콘 (Sik-Contract)
- **타겟**: 복잡한 법률 용어가 어려운 식당 사장님 및 아르바이트생
- **핵심 가치**: 토스(Toss) 스타일의 심플한 UX, AI를 통한 자동 조항 생성, 비대면 서명
- **플랫폼**: 모바일 웹/앱 (사업자용 / 근로자용 분리)

## 2. 주요 기능 및 프로세스

### 2.1 공통 사항 (로그인 및 진입)

- **구글 소셜 로그인**: Firebase Auth 등을 활용한 간편 인증
- **둘러보기 모드**: 회원가입 없이도 계약서 작성 체험 가능 (단, 저장 및 발송 시 가입 유도)
- **UI/UX 원칙**: 한 화면에 하나의 질문만 노출 (Step-by-Step), 큰 글씨와 명확한 버튼

### 2.2 [사업자용 앱] 기능

- **AI 계약서 생성**: 시작일, 시급, 근무 시간만 입력하면 AI가 최신 근로기준법에 맞는 표준계약서 초안 생성
- **계약 관리**: 진행 중, 완료된 계약서를 리스트 형태로 관리
- **근로자 초대**: 카카오톡 또는 링크 공유를 통해 근로자에게 계약 요청 발송

### 2.3 [근로자용 앱] 기능

- **계약서 확인**: 사장님이 보낸 계약 조건을 카드 슬라이드 형태로 확인
- **AI 조항 설명**: 어려운 법률 용어를 터치하면 AI가 쉬운 말로 풀어서 설명
- **간편 서명**: 모바일 화면에 직접 서명(Canvas) 후 즉시 완료

## 3. 기술 스택 (Technical Stack)

| 구분 | 기술 스택 | 비고 |
|------|-----------|------|
| Frontend | React (Vite) / PWA | 토스 스타일 UI 컴포넌트 라이브러리 활용 |
| Backend | Supabase Edge Functions | AI 연동을 위한 확장성 고려 |
| Database | Supabase (PostgreSQL) | 계약 정보 및 서명 이미지 경로 저장 |
| AI 모델 | OpenAI GPT-4o / Lovable AI | 계약서 초안 생성 및 조항 해석 |
| 인증 | Google OAuth 2.0 | Supabase Auth 활용 |

## 4. 상세 화면 설계 (UI Flow)

### Step 1: 온보딩 및 로그인

- 화면 1: 서비스 슬로건과 함께 "구글로 시작하기", "둘러보기" 버튼 배치

### Step 2: [사업자] 계약서 작성 (토스형 UX)

1. 질문: "사장님, 근로자의 성함은 무엇인가요?" (입력창)
2. 질문: "시급은 얼마로 정하셨나요?" (숫자 키패드 자동 활성화)
3. 질문: "근무 시작일은 언제인가요?" (날짜 선택)
4. 질문: "근무 시간을 알려주세요" (시간 선택)
5. AI 생성: 입력 정보를 바탕으로 "법적 효력이 있는 계약서를 생성 중입니다..." 애니메이션 후 미리보기 제공

### Step 3: [근로자] 확인 및 서명

1. 링크 접속: 로그인 후 받은 계약서 열람
2. 서명 창: 하단 고정 버튼 "확인했으며, 서명합니다" 클릭 시 캔버스 팝업
3. 완료: PDF 생성 및 양측 보관함 저장

## 5. 개발 단계별 마일스톤

### [1단계] 프론트엔드 구축 (UI/UX 중심)

- 토스 디자인 시스템(TDS) 스타일의 컴포넌트 제작 (버튼, 입력창, 프로그레스 바)
- 사업자용/근로자용 각 뷰(View) 및 네비게이션 구조 설계
- 둘러보기 기능을 위한 Mock Data 연동

### [2단계] 백엔드 구축 및 연결 (Data & AI)

- Supabase를 이용한 데이터베이스 설계
- OpenAI API 연동을 통한 실시간 계약서 생성 로직 구현
- PDF 라이브러리를 활용한 계약서 다운로드 기능 추가

## 6. 디자인 가이드라인

### 색상 팔레트

- **Primary**: 블루 계열 (#3182F6 토스 블루)
- **Background**: 깔끔한 화이트/라이트 그레이
- **Text**: 다크 그레이/블랙
- **Success**: 그린
- **Warning**: 옐로우
- **Error**: 레드

### UI 원칙

1. 한 화면에 하나의 질문만
2. 큰 글씨 (최소 18px)
3. 명확한 CTA 버튼
4. 부드러운 트랜지션 애니메이션
5. 프로그레스 인디케이터 활용
